## 架构模式

| 模式 | 说明 |
|------|------|
| **前后端分离** | Vue 3 前端 + Spring Boot 后端，通过 RESTful API 通信 |
| **三层架构** | Controller → Service → Mapper (MyBatis Plus) |
| **AOP 切面编程** | 使用 @ApiTest 注解 + Aspect 实现 Allure 报告自动生成 |
| **异步并发执行** | 通过 @Async + CompletableFuture + Semaphore 实现并发测试执行 |
| **POM 模式** | UI 测试采用 Page Object Model 设计模式 |
| **统一响应格式** | 所有 API 返回 Result<T> 统一包装类 |

## 系统边界

- **前端**: Vue 3 SPA，通过 `/api` 代理访问后端
- **后端**: Spring Boot 应用，端口 8080
- **数据库**: MySQL `test_platform` 数据库
- **缓存**: Redis (LocalDateTime 序列化配置)
- **测试引擎**: Apache HttpClient5 (API) + Selenium WebDriver (UI)
- **报告系统**: Allure 2.30 框架 (Java 生成 HTML 报告)

## 核心业务流程

### API 测试流程
```
用户创建项目 → 添加接口用例 → 单用例执行 / 批量并发执行
                                      ↓
                              TestReportAspect 拦截
                                      ↓
                            Allure 结果 JSON 生成
                                      ↓
                        存入 test_case_report 表
                                      ↓
                        查询/导出 Allure HTML 报告
```

### UI 测试流程
```
用户创建 UI 项目 → 添加 UI 用例(含步骤) → 单用例执行 / 批量并发执行
                                                ↓
                                    WebDriverFactory 创建驱动
                                                ↓
                                    BasePage 逐步骤执行操作
                                                ↓
                                    步骤断言验证 → 返回结果
```
