# 用户管理系统 - API 测试需求文档

## 1. 用户登录

### 1.1 用户登录接口
用户通过用户名和密码进行登录，获取访问令牌。

**接口说明：**
- 模块：用户认证
- URL：需要确认
- HTTP 方法：POST
- Content-Type：application/json

**请求参数：**
```json
{
  "username": "admin",
  "password": "123456"
}
```

**预期响应：**
- 成功时返回用户信息和 token
- 失败时返回错误码和提示

**注意事项：**
- 需要验证各种异常情况

---

## 2. 用户管理

### 2.1 获取用户列表
分页查询系统用户列表。

**接口说明：**
- 模块：用户管理
- HTTP 方法：需要确认
- 需要认证（Header 携带 token）

**查询参数：**
- page：页码，默认 1
- size：每页条数，默认 10
- keyword：搜索关键字（可选）

**预期响应：**
- 返回用户列表和分页信息
- 包含用户 ID、用户名、昵称、邮箱、状态、创建时间

### 2.2 创建用户
新增一个系统用户。

**接口说明：**
- 模块：用户管理
- URL：需要确认
- HTTP 方法：POST

**请求参数：**
```json
{
  "username": "zhangsan",
  "nickname": "张三",
  "email": "zhangsan@example.com",
  "phone": "13800138000",
  "password": "abc123",
  "role": "user"
}
```

**预期响应：**
- 成功返回新增用户 ID
- 用户名重复时应提示已存在

### 2.3 更新用户
修改用户信息。

**接口说明：**
- 模块：用户管理
- URL 格式：需要确认，可能需要传用户 ID

**请求参数：**
```json
{
  "nickname": "张三丰",
  "email": "zhangsanfeng@example.com",
  "phone": "13900139000"
}
```

**预期响应：**
- 成功返回更新后的用户信息

### 2.4 删除用户
删除指定用户。

**接口说明：**
- 模块：用户管理
- URL 格式：需要确认

**预期行为：**
- 删除成功返回确认信息
- 不存在的用户应返回错误
- 不能删除超级管理员

### 3.1 获取角色列表
查询系统所有角色。

## 4. 通用要求

### 4.1 认证方式
- 所有接口（除登录外）需要在请求头携带认证信息
- Token 过期时应返回 401

### 4.2 统一响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

### 4.3 需覆盖的测试场景
- 正常流程：正确的参数
- 参数校验：缺少必填字段、字段格式错误、参数超长
- 鉴权校验：未登录、token过期、无权限
- 数据不存在：查询/更新/删除不存在的记录
- 重复数据：创建重复用户名
